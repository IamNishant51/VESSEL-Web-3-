# Vessel

Solana-Native AI Agent Orchestrator & Marketplace

One-line description:
Vessel lets users mint autonomous AI agents as compressed NFTs on Solana devnet, orchestrate agent-to-agent tasks, and list, buy, or rent agents through a marketplace.

## Tech Stack

- Next.js 15 (App Router, TypeScript)
- React 19
- Zustand + localStorage persistence
- Solana Wallet Adapter
- Solana Web3.js + Metaplex Bubblegum (wallet-signed compressed NFT mint flow)
- Framer Motion
- Tailwind CSS
- Sonner (notifications)

## Setup

1. Clone repository

```bash
git clone <your-repo-url>
cd VESSEL
```

2. Install dependencies

```bash
npm install
```

3. Configure environment variables

```bash
cp .env.example .env.local
```

Required/optional variables are documented in `.env.example`.

4. Start local app

```bash
npm run dev
```

5. Open app

```text
http://localhost:3000
```

## Core Flow

1. Open the app and connect wallet.
2. Go to Forge:
	- Create an agent and mint compressed NFT (wallet-signed devnet transaction).
4. Go to Agents:
	- Open your agent detail page.
	- Use Chat tab to run tool calls.
	- Verify action increments reputation and total actions.
5. In Agent Detail, switch to Orchestra tab:
	- Select Agent B.
	- Submit orchestration prompt.
	- Verify chain shows delegation + x402 micropayment line + transaction explorer link.
6. Go to Marketplace:
	- List an agent for sale/rent.
	- Buy/rent from another wallet context if available.
	- Verify ownership/listing updates and transaction link.
7. Go to Dashboard:
	- Verify synced totals (actions, earnings, reputation signals) after refresh.

## Working Features

- Wallet connect/disconnect flow
- Global persisted state (agents, listings, stats)
- Agent minting flow with transaction metadata
- Agent run/chat tool execution with tx signature + explorer link
- Reputation and total action tracking per successful tool execution
- Agent treasury tracking
- Agent-to-agent orchestration workflow
- x402 micropayment simulation (0.001 USDC)
- Marketplace listing, buying, renting, ownership updates

## How To Use Core Flows

### Mint an Agent

1. Open Forge
2. Configure name, personality, tools, limits
3. Click mint action
4. Store updates with new agent, treasury, and stats defaults

### Run an Agent

1. Open Agents
2. Enter agent detail
3. Use Chat tab and send prompt
4. If tool executes:
	- Tx signature + explorer link returned
	- Reputation and total actions increment
	- Treasury debits micropayment for external tool call simulation

### Orchestrate Agents

1. Open agent detail for Agent A
2. Switch to Orchestra tab
3. Select Agent B and provide prompt
4. Workflow executes:
	- Agent A delegates to Agent B
	- x402 simulation transfers 0.001 USDC from A to B
	- Both agents update treasury/stat state
	- Full chain logged with explorer link

### Buy/Rent on Marketplace

1. List from Agents page
2. Open Marketplace listing detail
3. Buy or rent
4. Transaction signature + explorer link shown
5. Ownership/listing state syncs across pages and persists after refresh

## Error Handling Coverage

The app now surfaces clear messages for:

- wallet not connected
- listing not found
- attempting to buy/rent your own listing
- missing tools or missing allowed actions
- insufficient budget / invalid limits
- insufficient treasury for micropayment simulation
- API/runner failures with explicit error codes

## Runtime Notes

- Bubblegum minting requires `NEXT_PUBLIC_BUBBLEGUM_MERKLE_TREE` and `NEXT_PUBLIC_BUBBLEGUM_COLLECTION_MINT` in your environment.
- Marketplace settlement and orchestration treasury transfers remain app-level state transitions unless backed by your own on-chain programs.
